# Awesome-Embodied-Intelligence-Generalization
详情见：[中文 README](./README_ZH.md)
## 项目简介

这个仓库用于建设一个面向“具身智能泛化性”的论文索引与综述资料库。我们希望它不仅是一个论文列表，也能帮助大家回答三个更具体的问题：具身智能中的泛化到底泛化到哪里，现有方法通过什么机制获得泛化，以及现有 benchmark 是否真的支撑相应的泛化声明。

当前版本采用五层 RQ2 分类框架整理论文：数据层、表示层、模型层、学习层和系统层。每篇论文会被放入一个最贴近的主层次，同时用多个 tag 保留交叉机制。

## 当前协作任务

目前这个 repo 仍在整理和校对阶段，欢迎团队成员重点检查以下内容：

- **子类划分是否合理**：例如 D1-D5、R1-R5、M1-M5、L1-L5、S1-S5 的边界是否清楚，是否有需要合并、拆分或重命名的地方。
- **论文分类是否准确**：检查每篇论文的主层次、RQ2 tags、任务场景和泛化维度是否贴合论文真实贡献。
- **链接是否准确可用**：优先检查 Paper/arXiv、Project Website、GitHub Code 链接是否能打开，GitHub 链接是否指向真正的代码仓库。
- **论文是否值得保留**：扩充候选中可能包含 workshop、preprint 或相关性较弱的论文，欢迎标注需要删除、降级或人工复核的条目。
- **是否遗漏重要论文**：尤其欢迎补充 2022-2026 年顶会顶刊中与泛化、迁移、跨本体、sim-to-real、VLA、World Model、long-horizon embodied planning 相关的论文。

## 如何新增论文

推荐通过 Pull Request 或 Issue 新增论文。为了便于维护，请尽量提供以下信息：

| 字段 | 说明 |
|---|---|
| Title | 论文标题 |
| Year / Venue | 年份与会议/期刊，例如 `ICLR 2026`、`CoRL 2025`、`arXiv 2026` |
| Paper Link | arXiv、OpenReview、CVF、会议页面或期刊页面，至少提供一个 |
| Project / Code | 如果有项目主页或 GitHub 代码，请一并提供 |
| RQ2 Tags | 例如 `M1`, `L4`, `D3`，可多选 |
| Generalization Focus | 对应 RQ1 的维度，例如对象、环境、语言、任务、技能、物理、本体、迁移、长程、安全 |
| Short Note | 一句话说明这篇论文为什么与具身泛化有关 |

如果不确定如何分类，可以先只提交论文标题、链接和简短理由，我们会一起讨论归类。
